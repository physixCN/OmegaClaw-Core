# v0.01a Module Profile Recommendations

This note separates the reusable out-of-box core profile from the fuller Live
Omega situated profile. It is a recommendation, not a runtime loader change yet.

## Principle

`default_enabled=true` should mean one of three things:

- the capability is part of the substrate's core cognition or continuity;
- the capability is safe, local, and useful without private credentials;
- the capability is a reviewer demo needed to understand the architecture.

Everything that requires private credentials, device access, paid providers,
large dependencies, third-party network state, or local personal data should be
module-packaged but default-off in a clean public profile.

## Suggested Clean Core Profile

| Module | Suggested clean default | Why |
|---|---:|---|
| `scratch_space` | on | Core working-memory organ; local, symbolic, traceable. |
| `channel_router` | on | Generic control surface; useful for routing without choosing a private channel. |
| `channel_mock` | on | Safe local demo/testing channel. |
| `web_search` | on if key configured | External knowledge acquisition is fundamental, but must fail visibly without credentials. |
| `assume` | on | Core research contribution: symbolic assumption space plus numeric mirror. |
| `scheduler_reminders` | on | Small symbolic continuity/time organ. |
| `sense_router` | on | Generic observation router; route declarations are symbolic and fail closed. |
| `vm_policy` | on | Boundary/immune introspection, no mutation by default. |

## Live Omega Profile

Live Omega may intentionally enable more modules because she has configured
channels, senses, health apps, media providers, AgentVerse, and local device
state. That is a situated deployment profile, not the clean public default.

## Recommended Repo Shape

- Keep every module packaged and documented.
- Add explicit loader profiles before public v0.01a if time allows:
  - `modules/profiles/core.metta`
  - `modules/profiles/reviewer-demo.metta`
  - `modules/profiles/live-situated.example.metta`
- Let `modules/loader.metta` import the clean core/reviewer profile in the
  public repo, while Live can import its private situated profile.
