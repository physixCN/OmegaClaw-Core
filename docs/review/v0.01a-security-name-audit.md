# v0.01a Name And Security Audit

This note records the current reviewer-facing boundary for the private readiness branch. It is intentionally plain: reviewers should be able to see what was checked, what is included, and what is deliberately outside the shareable core.

## Scope

Checked tracked source, docs, tests, module metadata, and review artifacts for:

- private names and deployment labels;
- token-like values and private key blocks;
- runtime auth/session files;
- local webhost/public-site state;
- module declarations that mention secrets.

## Result

- No private deployment credentials are tracked.
- No WhatsApp auth/session directories are tracked.
- No local webhost runtime state is tracked.
- `memory/prompt.txt` is tracked as the generic seed prompt; runtime memories, histories, web state, generated media, ledgers, and channel traces are ignored.
- Secret-like terms that remain in tracked files are scanner regexes or environment-variable names, not secret values.
- Channel, provider, device, and body-app secrets are declared in `module.toml`/docs as runtime configuration.

## Runtime State Boundary

Ignored runtime paths include:

- `memory/*.metta`, except curated `memory/prompt.txt`;
- `memory/*.json`, `memory/*.jsonl`, ledgers, databases, logs, and backups;
- `memory/web/`, `memory/web/public/`, `memory/inbox/`, `memory/outbox/`, and generated media;
- module-owned WhatsApp bridge `node_modules/` and `auth*/` session directories;
- `.env`, local backup files, and pulled dependency repos.

The working tree used for private development may still contain ignored runtime files locally. Those are not part of the repository artifact and should not be copied into review bundles.

## Names

The public/private readiness artifact avoids naming private deployment people, family members, domains, phone numbers, or channels. Generic words such as `family`, `operator`, `Patient`, and synthetic `123@lid`/`523@test` values appear in tests and examples only.

## Module Secrets

Modules may declare required runtime secrets, for example provider API keys, Mattermost/Telegram/Slack tokens, Home Assistant tokens, or WhatsApp auth directories. These declarations are intentional and make dependencies visible; values must be supplied by the deployment environment and must not be committed.

## Reviewer Command

```bash
PYTHONPATH=src python3 docs/review/review_audit.py
```

The current audit gate checks secret-like patterns, patch boundaries, runtime memory import boundaries, patch applicability, and review-surface coverage.
